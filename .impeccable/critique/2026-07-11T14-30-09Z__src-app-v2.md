# Anti-Patterns Verdict

**Not AI slop — genuinely.** Every absolute ban checked and refused: no side-stripe borders, no gradient text, no hero-metric template, no identical card grids, no 01/02/03 scaffolding, no repeated uppercase eyebrows, radii 8–12px, display letter-spacing -0.035em (inside the floor), clamp max exactly 6rem. Color is committed (indigo → blue field → white → vermilion drench per section), not beige-timid. One deliberate glassmorphism instance (nav pill) with @supports and reduced-transparency fallbacks.

**Deterministic scan**: source scan clean (exit 0, zero findings). Runtime DOM overlay flagged: tight-leading ×4 (1.25 line-height on discipline statements), line-length ×3 (~96ch paragraphs), tiny-text ×2 (11.5px utility labels), hero-eyebrow-chip ×1, single-font ×1, clipped-overflow-container ×1. False positives among these: single-font (Archivo is a committed single-family system, allowed), hero eyebrow (one kicker is voice, not scaffold grammar), clipped-overflow (position:fixed header escapes overflow:clip; benign). The leading/line-length/tiny-text hits are real micro-legibility debt.

**Honest caveat (second-order)**: the site does live in a recognizable lane — huge tight grotesk, asymmetric grid, saturated per-section color fields. Executed with enough commitment that it reads designed, not generated, but it is not yet unmistakable.

# Overall Impression

Clean, credible, accessibility done for real — top-decile for a portfolio. The peak is the Operating Range section (drifting indigo light field + grain + sticky heading). The gap to "stunning" is not more decoration; it's that the two most prominent visual modules — the hero portrait and the first project image — are the two weakest-executed surfaces on the page.

# What's Working

1. **Accessibility is real, not claimed.** All contrast pairs pass AA with margin (body-on-canvas 8.95:1, white-on-blue 5.27:1, white-on-vermilion 5.84:1, placeholder 7.93:1). Skip link, 3px :focus-visible, native dialog, forced-colors, reduced-transparency, and comprehensive prefers-reduced-motion coverage.
2. **Color commitment.** One saturated field per section; grain only where color is the material; reading surfaces stay clean. Palette IS the voice.
3. **Projects as physical artifacts.** Alternating layout, hard "artifact edge" shadow, scroll-driven tilt-in. The AI Medical Agent screenshot on indigo is the model for how all three should look.

# Priority Issues

**[P1] First project shows no product.** The AI Companion image renders as an orange gradient with a barely-visible device outline. The first exhibit under the headline "Evidence over claims" contains no evidence — directly undercuts the site's core goal. Fix: real screenshot with the same artifact-edge treatment as AI Medical Agent. Suggested command: $impeccable polish

**[P1] Desktop section-heading dead space.** The 0.65fr/1.3fr/1fr heading grid with align-items:end drops label + description to the baseline of a 5.5rem headline, leaving a large empty band on Work and Journey at 1440px. Reads unfinished, not airy. Fix: top-align the meta column or cap the heading row. Suggested command: $impeccable layout

**[P2] Hero portrait mixes media registers.** Default state is a photo on a grey background floating inside the orange gradient field; the reveal is an illustration cut to transparency. The signature module is the weakest surface on the page. Fix: cut the confident portrait to transparency so both states sit directly on the gradient; unify treatment. Suggested command: $impeccable polish

**[P2] Contact form is emotionally and technically bare at the highest-stakes moment.** No reassurance line (privacy, response expectation), and handleSubmit has no try/catch — if the server action throws (network drop), the rejection is unhandled and the user gets no feedback. Fix: one reassurance line under the button + wrap the submit in try/catch with an error status. Suggested commands: $impeccable clarify + $impeccable harden

**[P2] Micro-legibility debt (detector-confirmed).** 11.5px (0.72rem) utility labels across hero context, meta rows, and footer; ~96-char line lengths on three paragraphs; 1.25 line-height on discipline statements. Individually small, collectively the difference between crisp and strained. Fix: raise utility labels to ≥0.78rem, cap offending paragraphs at ~70ch, leading to 1.3. Suggested command: $impeccable typeset

# Persona Red Flags

**Riley (stress tester)**: Scroll-spy rootMargin (-34%/-56%) leaves dead bands at page top/bottom where no nav item is active and the lens vanishes. The unguarded submit handler is the bigger catch — a thrown server action leaves the form silent.

**Casey (mobile, one thumb)**: Menu trigger sits top-right, the hardest one-thumb zone, and it's the only route to Contact. Both portrait PNGs load eagerly (one priority) at 34rem for a swap Casey may never trigger — heavy on slow connections.
